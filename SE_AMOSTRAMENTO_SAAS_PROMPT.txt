# SE Amostramento Pro — Plataforma SaaS EdTech B2B

## OBJETIVO
Criar do zero uma plataforma SaaS EdTech B2B chamada **SE Amostramento Pro** — um produto
de capacitação em Comunicação Corporativa baseado no Método SE Amostramento Profissional.
A plataforma vende acesso a trilhas de comunicação, workshops e materiais exclusivos do método
para profissionais autônomos, equipes e empresas de todos os portes.

Não há nenhum projeto base. Construa tudo do zero.

---

## STACK OBRIGATÓRIA

- **Frontend**: React 18 + TypeScript + Vite + TailwindCSS v4 + shadcn/ui +
  TanStack Router (file-based) + TanStack Query + Framer Motion + Lucide React
  (ícones exclusivamente) + Sonner (toasts) + Recharts (gráficos) + Zod
  (validação)
- **Backend**: Supabase (PostgreSQL + Auth + Edge Functions + Storage + Realtime)
- **Pagamentos**: Stripe (Hosted Checkout — card e PIX)
- **Email**: Resend
- **Regras rígidas de código**:
  - Zero `any` em TypeScript — use `unknown` com type guard
  - Zod em todos os formulários e Edge Functions
  - Nenhuma dependência nova além das listadas acima
  - Mobile-first obrigatório (375px como base)
  - WCAG 2.1 AA em todos os componentes
  - ARIA correto em todos os elementos interativos
  - Lucide React como única fonte de ícones
  - Nunca editar `routeTree.gen.ts` manualmente

## DESIGN SYSTEM — SE AMOSTRAMENTO PRO

### Identidade Visual
O design segue a identidade do Método SE Amostramento Profissional: visual escuro,
premium e corporativo com gradiente roxo-azul, tipografia bold e personalidade de marca
forte. O tema padrão é **dark mode** com opção de light mode.

### Regras de uso
- CTA primário: fundo `#7B2FBE`, texto `#FFFFFF`, hover `#6B21A8`
- CTA secundário: borda `#2979FF`, texto `#2979FF`, hover com fundo `rgba(41,121,255,0.1)`
- Badge premium / plano Elite: fundo `#BF5AF2`, texto `#FFFFFF`
- Gradiente de marca em banners, heroes e CTAs de destaque: `var(--gradient-brand)`
- Hero sections: `var(--gradient-hero)` sobre fundo escuro
- Cards no dark mode: `#18182A` com borda `#2A2A40`
- NUNCA usar texto muted (`#7C7C9C`) em fundo escuro sem tamanho mínimo 14px
- Tipografia: **Space Grotesk** (sans-serif) para UI e body; **Bebas Neue** ou **Oswald** para headings grandes e logo text (bold, condensed, uppercase — refletindo a identidade visual do método)
- Todos os botões primários com border-radius de 8px; não arredondar excessivamente
- Inputs com borda `#2A2A40` e focus ring `#7B2FBE`

### Efeitos visuais de marca
- Glow roxo sutil em elementos de destaque: `box-shadow: 0 0 20px rgba(123,47,190,0.4)`
- Glow azul em hover de elementos accent: `box-shadow: 0 0 16px rgba(41,121,255,0.35)`
- Gradiente de fundo nos heroes com partículas/ruído sutil simulando a identidade do PDF
- Cards com borda `1px solid #2A2A40` e hover com `1px solid #7B2FBE`

## ENTIDADES DO NEGÓCIO

### Tipos de conta (enum `account_type`)
- `individual` — Profissional autônomo / Pessoa Física
- `startup` — Equipe pequena com até 15 usuários
- `empresa` — Empresa com até 50 usuários
- `corporativo` — Empresa sem limite de usuários (plano enterprise)

### Papéis (enum `user_role`)
- `admin` — Equipe interna da SE Amostramento (acesso total)
- `org_admin` — Gestor da empresa cliente (gerencia membros da própria org)
- `membro` — Usuário final que consome conteúdo

### Planos de assinatura
- **Solo** (individual): 1 usuário, acesso ao Método SE básico + trilhas essenciais
- **Team** (startup): até 15 usuários, método completo + workshops gravados + materiais
- **Empresa** (empresa): até 50 usuários, tudo + simulações práticas + relatórios de evolução
- **Corporativo** (corporativo): usuários ilimitados, tudo + consultoria dedicada + customização de acesso por departamento + eventos ao vivo

```sql
-- Extensões
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS "pgcrypto";

-- Enums
CREATE TYPE account_type AS ENUM ('individual', 'startup', 'empresa', 'corporativo');
CREATE TYPE user_role AS ENUM ('admin', 'org_admin', 'membro');
CREATE TYPE subscription_status AS ENUM ('active', 'trialing', 'past_due', 'canceled', 'paused');
CREATE TYPE content_type AS ENUM ('video', 'pdf', 'slide', 'quiz', 'simulacao', 'live');
CREATE TYPE request_status AS ENUM ('pending', 'approved', 'rejected');

-- Organizações (empresas clientes)
CREATE TABLE organizations (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name            TEXT NOT NULL,
  slug            TEXT UNIQUE NOT NULL,
  account_type    account_type NOT NULL DEFAULT 'individual',
  logo_url        TEXT,
  website         TEXT,
  cnpj            TEXT,
  max_members     INTEGER NOT NULL DEFAULT 1,
  is_active       BOOLEAN NOT NULL DEFAULT true,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Perfis de usuário (vinculados ao Supabase Auth)
CREATE TABLE profiles (
  id              UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  org_id          UUID REFERENCES organizations(id) ON DELETE SET NULL,
  role            user_role NOT NULL DEFAULT 'membro',
  full_name       TEXT NOT NULL,
  email           TEXT NOT NULL,
  avatar_url      TEXT,
  bio             TEXT,
  department      TEXT,
  job_title       TEXT,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Assinaturas
CREATE TABLE subscriptions (
  id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id                UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  stripe_customer_id    TEXT,
  stripe_subscription_id TEXT,
  plan_name             TEXT NOT NULL,
  account_type          account_type NOT NULL,
  status                subscription_status NOT NULL DEFAULT 'trialing',
  current_period_start  TIMESTAMPTZ,
  current_period_end    TIMESTAMPTZ,
  created_at            TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at            TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Solicitações de acesso
CREATE TABLE account_requests (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  full_name     TEXT NOT NULL,
  email         TEXT NOT NULL,
  org_name      TEXT,
  account_type  account_type NOT NULL DEFAULT 'individual',
  job_title     TEXT,
  message       TEXT,
  status        request_status NOT NULL DEFAULT 'pending',
  reviewed_by   UUID REFERENCES profiles(id),
  reviewed_at   TIMESTAMPTZ,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Trigger: atualiza updated_at automaticamente
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER LANGUAGE plpgsql AS $$
BEGIN NEW.updated_at = now(); RETURN NEW; END; $$;

CREATE TRIGGER trg_organizations_updated_at
  BEFORE UPDATE ON organizations FOR EACH ROW EXECUTE FUNCTION update_updated_at();

CREATE TRIGGER trg_profiles_updated_at
  BEFORE UPDATE ON profiles FOR EACH ROW EXECUTE FUNCTION update_updated_at();

-- Trigger: cria profile automaticamente ao criar usuário no Auth
CREATE OR REPLACE FUNCTION handle_new_user()
RETURNS TRIGGER LANGUAGE plpgsql SECURITY DEFINER AS $$
BEGIN
  INSERT INTO profiles (id, email, full_name)
  VALUES (
    NEW.id,
    NEW.email,
    COALESCE(NEW.raw_user_meta_data->>'full_name', split_part(NEW.email, '@', 1))
  )
  ON CONFLICT (id) DO NOTHING;
  RETURN NEW;
EXCEPTION WHEN OTHERS THEN RETURN NEW;
END; $$;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users FOR EACH ROW EXECUTE FUNCTION handle_new_user();
```

-- Trilhas de comunicação corporativa
CREATE TABLE trails (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title         TEXT NOT NULL,
  description   TEXT,
  thumbnail_url TEXT,
  category      TEXT NOT NULL DEFAULT 'comunicacao', -- comunicacao | oratoria | lideranca | apresentacao | negociacao
  level         TEXT NOT NULL DEFAULT 'essencial',   -- essencial | intermediario | avancado | master
  duration_min  INTEGER DEFAULT 0,
  is_published  BOOLEAN NOT NULL DEFAULT false,
  is_featured   BOOLEAN NOT NULL DEFAULT false,
  order_index   INTEGER NOT NULL DEFAULT 0,
  instructor_name TEXT,
  instructor_bio  TEXT,
  instructor_avatar_url TEXT,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Módulos dentro de uma trilha
CREATE TABLE trail_modules (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  trail_id    UUID NOT NULL REFERENCES trails(id) ON DELETE CASCADE,
  title       TEXT NOT NULL,
  description TEXT,
  order_index INTEGER NOT NULL DEFAULT 0,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Conteúdos (aulas, PDFs, slides, quizzes, simulações, lives)
CREATE TABLE contents (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  description   TEXT,

  storage_path  TEXT,
  external_url  TEXT,
  duration_min  INTEGER DEFAULT 0,

  is_published  BOOLEAN NOT NULL DEFAULT false,
  is_preview    BOOLEAN NOT NULL DEFAULT false, -- amostra gratuita
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Sessões ao vivo / workshops (lives)
CREATE TABLE live_sessions (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  trail_id        UUID REFERENCES trails(id) ON DELETE SET NULL,
  title           TEXT NOT NULL,

  recording_url   TEXT,
  is_published    BOOLEAN NOT NULL DEFAULT false,

CREATE TABLE org_content_access (

  expires_at  TIMESTAMPTZ,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(org_id, content_id)
);

CREATE TRIGGER trg_trails_updated_at

### 004_rls_policies.sql
```sql
-- RLS: organizations
ALTER TABLE organizations ENABLE ROW LEVEL SECURITY;

CREATE POLICY "org_members_see_own_org" ON organizations
  FOR SELECT TO authenticated
  USING (id = (SELECT org_id FROM profiles WHERE id = auth.uid()));

CREATE POLICY "admin_full_access_orgs" ON organizations
  FOR ALL TO authenticated
  USING ((SELECT role FROM profiles WHERE id = auth.uid()) = 'admin');

-- RLS: profiles
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users_see_own_profile" ON profiles
  FOR SELECT TO authenticated
  USING (id = auth.uid());

CREATE POLICY "users_see_same_org_profiles" ON profiles
  FOR SELECT TO authenticated
  USING (
    org_id IS NOT NULL AND
    org_id = (SELECT org_id FROM profiles WHERE id = auth.uid())
  );

CREATE POLICY "users_update_own_profile" ON profiles
  FOR UPDATE TO authenticated
  USING (id = auth.uid());

CREATE POLICY "admin_full_access_profiles" ON profiles
  FOR ALL TO authenticated
  USING ((SELECT role FROM profiles WHERE id = auth.uid()) = 'admin');

-- RLS: account_requests
ALTER TABLE account_requests ENABLE ROW LEVEL SECURITY;

CREATE POLICY "anon_can_request_access" ON account_requests
  FOR INSERT TO anon, authenticated
  WITH CHECK (true);

CREATE POLICY "admin_manages_requests" ON account_requests
  FOR ALL TO authenticated
  USING ((SELECT role FROM profiles WHERE id = auth.uid()) = 'admin');

-- RLS: trails
ALTER TABLE trails ENABLE ROW LEVEL SECURITY;

CREATE POLICY "members_see_accessible_trails" ON trails
  FOR SELECT TO authenticated
  USING (
    is_published = true AND (
      (SELECT role FROM profiles WHERE id = auth.uid()) = 'admin'
      OR is_preview = true
      OR EXISTS (
        SELECT 1 FROM org_trail_access ota
        JOIN profiles p ON p.org_id = ota.org_id
        WHERE p.id = auth.uid() AND ota.trail_id = trails.id
        AND (ota.expires_at IS NULL OR ota.expires_at > now())
      )
    )
  );

CREATE POLICY "admin_full_access_trails" ON trails
  FOR ALL TO authenticated
  USING ((SELECT role FROM profiles WHERE id = auth.uid()) = 'admin');

-- RLS: contents
ALTER TABLE contents ENABLE ROW LEVEL SECURITY;

CREATE POLICY "members_see_accessible_contents" ON contents
  FOR SELECT TO authenticated
  USING (
    is_published = true AND (
      (SELECT role FROM profiles WHERE id = auth.uid()) = 'admin'
      OR is_preview = true
      OR EXISTS (
        SELECT 1 FROM org_content_access oca
        JOIN profiles p ON p.org_id = oca.org_id
        WHERE p.id = auth.uid() AND oca.content_id = contents.id
        AND (oca.expires_at IS NULL OR oca.expires_at > now())
      )
      OR EXISTS (
        SELECT 1 FROM org_trail_access ota
        JOIN profiles p ON p.org_id = ota.org_id
        JOIN trail_modules tm ON tm.trail_id = ota.trail_id
        WHERE p.id = auth.uid() AND tm.id = contents.module_id
        AND (ota.expires_at IS NULL OR ota.expires_at > now())
      )
    )
  );

CREATE POLICY "admin_full_access_contents" ON contents
  FOR ALL TO authenticated
  USING ((SELECT role FROM profiles WHERE id = auth.uid()) = 'admin');

-- RLS: user_progress
ALTER TABLE user_progress ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users_own_progress" ON user_progress
  FOR ALL TO authenticated
  USING (user_id = auth.uid());

CREATE POLICY "org_admin_see_org_progress" ON user_progress
  FOR SELECT TO authenticated
  USING (
    (SELECT role FROM profiles WHERE id = auth.uid()) IN ('admin', 'org_admin')
    AND user_id IN (
      SELECT id FROM profiles WHERE org_id = (
        SELECT org_id FROM profiles WHERE id = auth.uid()
      )
    )
  );

-- RLS: certificates

CREATE POLICY "users_own_certificates" ON certificates FOR SELECT TO authenticated USING (user_id = auth.uid());
CREATE POLICY "admin_manages_certificates" ON certificates FOR ALL TO authenticated
  USING ((SELECT role FROM profiles WHERE id = auth.uid()) = 'admin');

-- RLS: user_scores

CREATE POLICY "users_own_scores" ON user_scores FOR ALL TO authenticated USING (user_id = auth.uid());
CREATE POLICY "org_can_see_scores" ON user_scores FOR SELECT TO authenticated

-- RLS: content_notes
ALTER TABLE content_notes ENABLE ROW LEVEL SECURITY;
CREATE POLICY "users_own_notes" ON content_notes FOR ALL TO authenticated USING (user_id = auth.uid());

-- RLS: live_sessions

ALTER TABLE live_registrations ENABLE ROW LEVEL SECURITY;
CREATE POLICY "users_own_registrations" ON live_registrations FOR ALL TO authenticated USING (user_id = auth.uid());
CREATE POLICY "admin_see_all_registrations" ON live_registrations FOR SELECT TO authenticated
  USING ((SELECT role FROM profiles WHERE id = auth.uid()) = 'admin');

-- RLS: audit_log
ALTER TABLE audit_log ENABLE ROW LEVEL SECURITY;
CREATE POLICY "admin_sees_audit" ON audit_log FOR SELECT TO authenticated
  USING ((SELECT role FROM profiles WHERE id = auth.uid()) = 'admin');
CREATE POLICY "system_inserts_audit" ON audit_log FOR INSERT TO authenticated WITH CHECK (true);
```

## EDGE FUNCTIONS SUPABASE

### `provision-access` (POST)
Chamada quando admin aprova uma solicitação.
1. Valida JWT (admin apenas)
2. Cria organização em `organizations`
3. Cria usuário no Supabase Auth com senha temporária
4. Cria profile com `org_id` e `role = 'org_admin'`
5. Envia email de boas-vindas via Resend com senha temporária e link para a plataforma
6. Atualiza `account_requests` status para `approved`
7. Insere em `audit_log`

```typescript
const schema = z.object({
  request_id:   z.string().uuid(),
  org_name:     z.string().min(2),
  account_type: z.enum(['individual', 'startup', 'empresa', 'corporativo']),
  plan_name:    z.string(),
})
```

### `manage-member` (POST)
Operações de membros que requerem service_role:
- `action: 'invite'` — cria usuário e vincula à org
- `action: 'remove'` — desvincula da org e desativa
- `action: 'change_role'` — altera role do profile

### `issue-certificate` (POST)
Emissão de certificado ao concluir trilha:
1. Valida conclusão da trilha (user_trail_progress.completed = true)
2. Gera registro em `certificates`
3. Envia email via Resend com link do certificado
4. Insere conquista "Certificado SE" em `user_achievements` se aplicável

```typescript
const schema = z.object({
  user_id:  z.string().uuid(),
  trail_id: z.string().uuid(),
})
```

## REGRAS FINAIS

1. Nunca editar `routeTree.gen.ts` — o TanStack Router plugin regenera automaticamente
2. Usar `src/lib/supabase.ts` como cliente singleton do Supabase
3. Todos os formulários com React Hook Form + Zod + `zodResolver`
4. Toasts via Sonner (não alert/window.confirm)
5. Todos os modais via `Dialog` do shadcn/ui com focus trap
6. Skeleton loading dark em toda listagem que aguarda dados do Supabase
7. Nenhum `console.log` em produção — use `console.warn` apenas para debug
8. Imports de ícones sempre do Lucide React, nunca de outra lib
9. Paleta SE Amostramento aplicada via CSS variables — não hardcode hex inline
10. `AuthGuard` em todas as rotas protegidas antes de qualquer fetch
11. **Dark mode é o padrão** — usar `class="dark"` no `<html>` por padrão; light mode como alternativa via toggle
12. Fontes: carregar **Space Grotesk** e **Oswald** do Google Fonts para reproduzir a identidade visual bold do Método SE
13. Todos os backgrounds principais usando `var(--color-bg)` (#080810) — nunca branco como padrão
14. Cards com borda roxa no hover: `hover:border-[#7B2FBE]` ou via CSS var
